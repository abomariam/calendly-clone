# Task 09: Build Booking Success Screen

## Goal

Create the scheduled confirmation screen shown after a successful booking.

## Steps

- Build `BookingSuccess` using event data and booking response data.
- Show a clear scheduled confirmation message.
- Show event name, invitee details where useful, formatted date/time range, selected timezone, and description.
- Omit unsupported screenshot actions such as opening an invitation.
- Keep the success screen visually close to the screenshot.

## Acceptance Criteria

- Successful bookings render a confirmation screen without refetching event data.
- The confirmation time is displayed in the invitee timezone.
- The layout works on desktop and mobile.
